Fresh identity, fully isolated from your personal accounts.

  3. Add your SSH public key to the new GitHub account
  4. Create new private repo named donna — leave it empty

  1. Go to GlobalCan/donna → Settings
  2. Scroll to Danger Zone → Transfer ownership
  3. Target: the new bot-ops account
  4. Confirm with the repo name

bash

5
Generate the age keypair 2 min
›

Used to encrypt secrets at rest with sops.

bash
mkdir -p ~/.donna
age-keygen -o ~/.donna/age.key
grep 'public key:' ~/.donna/age.key

Copy the age1... public key shown in the last command output.

Now edit .sops.yaml in the repo root — replace age1REPLACE_WITH_YOUR_PUBLIC_KEY_HERE with your public key.

bash
git add .sops.yaml
git commit -m "Add age public key for sops encryption"
git push
Guard the private key. ~/.donna/age.key never leaves your laptop and the droplet. If you lose it, every secret in secrets/*.enc.yaml is unreadable.
age key generated, sops.yaml updated →

Encrypt the first secrets file 5 min
›

Put your real keys into an encrypted YAML that ships in the repo safely.

bash
cat > /tmp/plain.yaml <<'EOF'
DISCORD_BOT_TOKEN=mfa.xxx
DISCORD_ALLOWED_USER_ID=123456789012345678
ANTHROPIC_API_KEY=sk-ant-xxx
TAVILY_API_KEY=tvly-xxx
VOYAGE_API_KEY=pa-xxx
EOF

sops -e /tmp/plain.yaml > secrets/prod.enc.yaml
rm /tmp/plain.yaml

git add secrets/prod.enc.yaml
git commit -m "Initial encrypted secrets"
git push
Verify: sops -d secrets/prod.enc.yaml should print your keys in plaintext (only you can decrypt — uses the private key from step 5).
Check: make sure /tmp/plain.yaml is deleted. The whole point is no plaintext on disk.
Secrets encrypted, committed, plaintext deleted →
7
Harden the droplet 10 min
›

Run the hardening script as root on the fresh droplet. It creates the bot user, locks SSH, enables ufw + fail2ban, installs Docker, sops, age.

bash
# Replace DROPLET_IP with your actual IP
scp scripts/harden-droplet.sh root@DROPLET_IP:/root/
ssh root@DROPLET_IP bash /root/harden-droplet.sh

After hardening, copy the age key to the droplet and lock it down:

bash
scp ~/.donna/age.key root@DROPLET_IP:/etc/bot/age.key
ssh root@DROPLET_IP "chown bot:bot /etc/bot/age.key && chmod 600 /etc/bot/age.key"
⚠️ Check permissions. sops will silently fail at container start if /etc/bot/age.key isn't chown bot:bot and chmod 600. #2 most common gotcha.

Update ~/.ssh/config on your laptop:

ssh config
Host donna
  HostName DROPLET_IP
  User bot
  IdentityFile ~/.ssh/id_ed25519
  ServerAliveInterval 60

Test: ssh donna should land you on the droplet as the bot user.

Droplet hardened, age key in place →

First deploy 15 min
›

SSH to the droplet as bot and bootstrap the deployment.

bash
ssh donna

# Clone the repo on the droplet
git clone git@github.com:<bot-ops-user>/donna.git
cd donna

# First-time: build the image locally (GHA may not have run yet)
docker build -t ghcr.io/<bot-ops-user>/donna:latest .

# Or: pull the prebuilt image (if GHA already pushed one)
# docker login ghcr.io -u <bot-ops-user>
# docker compose pull

# Create .env (runtime secrets come from sops, but compose needs the file present)
cp .env.example .env

docker compose up -d
docker compose exec bot alembic upgrade head
sudo systemctl enable --now donna-update.timer
First-deploy chicken-and-egg: the GHCR image won't exist until a git push to main triggers the workflow. Either build locally on the droplet (fast) or push a commit and wait for GHA (cleaner).
Verify: docker compose ps shows three healthy containers (bot, worker, phoenix). Logs: docker compose logs bot --tail 50.
Three containers up, migrations applied →

Teach your first scope 10 min
›

Start with a public-domain author — clean text, no edition ambiguity. Mark Twain's Huckleberry Finn from Project Gutenberg is ideal.

bash
# On laptop or droplet

    --source-type book \
    --title "Adventures of Huckleberry Finn" \
    --copyright-status public_domain \
    --publication-date 1884-12-10

Then in Discord, query the new scope:

discord
/ask scope:author_twain question:What did Twain think about formal education?
Expected: response in Twain's voice, grounded in retrieved chunks, with citations. If the corpus doesn't cover the topic, grounded mode will refuse cleanly: "I don't have material from author_twain on this topic."

Set up your first scheduled task 2 min
›

In Discord:

discord
/schedule cron_expr:0 8 * * * task:morning brief: what's new in AI today?

Cron is UTC. 0 8 * * * = 8am UTC every day. Adjust to your timezone.

Verify it scheduled:

discord
/schedules

Or from the CLI on either machine:

bash
botctl schedule list
Cron is the only proactive trigger in v1. Watchers, reflections, and self-scheduled follow-ups are v2 — after the eval harness and taint tracking have proven themselves under real use.
