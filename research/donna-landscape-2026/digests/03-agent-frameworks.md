## Three patterns to steal

1. **Pluggable per-step checkpointing with replay/fork.** (a) Every step writes a checkpoint into a backend you choose; runs can resume across process restarts and fork from any point. (b) LangGraph v1, MS Agent Framework 1.0 (superstep), Pydantic AI graph (`SimpleStatePersistence`/`FullStatePersistence` + Temporal), Claude Agent SDK (`SessionStore` + file checkpointing). (c) Donna's "oracle, not scholar" stance requires the operator to drill into evidence and re-run reasoning under different assumptions — checkpoint+fork is the mechanism, and SQLite-backed checkpoints satisfy the self-host tenet. (d) https://docs.langchain.com/oss/python/releases/langgraph-v1
2. **MCP hygiene primitives at the spawn boundary.** (a) Filter env to subprocesses (only PATH/HOME/USER/LANG/etc. — strip API keys), OSV malware scan on every npx/uvx-spawned MCP server, per-server tool allowlist, sanitise tool error strings before returning to the LLM. (b) Hermes Agent leads; Claude Agent SDK adds tool annotations (`readOnlyHint`/`destructiveHint`/`idempotentHint`/`openWorldHint`). (c) Donna's threat model assumes prompt injection is the default state of the open web; these are the most defensible primitives shipping today and directly map onto Donna's tool-sandboxing tenet. (d) https://hermes-agent.nousresearch.com/docs/user-guide/security
3. **Type-validated structured output with retry-on-failure.** (a) Tool/agent outputs declared as Pydantic models; framework constrains decoding and re-prompts on validation failure. (b) Pydantic AI (native), OpenAI Agents SDK Guardrails, Claude Agent SDK tool annotations. (c) Donna needs to render labeled fact vs. labeled inference distinctly — typed outputs with explicit "claim" vs. "inference" fields make the legibility guarantee enforceable at the schema layer rather than at the prompt layer. (d) https://ai.pydantic.dev/

## Three patterns to avoid

1. **Multi-agent conversation as the default primitive.** (a) Group-chat / role-played crews as the framework's first-class shape. (b) AutoGen, AG2, CrewAI. (c) The original creators forked AutoGen as AG2 in Nov 2024 over governance disputes; Microsoft's response (the v0.4 rewrite, then Agent Framework) put AutoGen into maintenance mode by Sep 2025. The conversation/crew primitive is overkill for a single-user oracle and the ecosystem has fragmented three ways. (d) https://github.com/ag2ai/ag2 + https://github.com/microsoft/autogen (maintenance-mode README, last release `v0.7.5` 2025-09-30)
2. **Forced consolidation migrations of vendor frameworks.** (a) Vendor announces v2 ≡ "different product, migrate now." (b) Microsoft Agent Framework absorbing both AutoGen and Semantic Kernel — second forced migration in 18 months for AutoGen users; first for SK users. (c) Donna cannot afford to re-platform a v0.5 personal assistant when its substrate gets renamed and re-shaped. Prefer LangGraph v1's explicit zero-breaking-change stability commitment over Microsoft's lineage. (d) https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/
3. **In-process plugin trust with no sandbox boundary.** (a) Tools/plugins execute inside the host process with the agent's full env (.env, OAuth tokens, shell). (b) Default for Semantic Kernel, AutoGen, LangGraph, LlamaIndex, MS Agent Framework. (c) The OpenClaw / ClawHavoc skill-store post-mortem (Jan–Feb 2026) is the canonical demonstration that in-process plugin trust + extension marketplace = supply-chain malware vector — 341→1,184 malicious skills, Atomic macOS Stealer payloads, weeks of dwell time. Donna must not inherit this default. (d) https://www.koi.ai/blog/clawhavoc-341-malicious-clawedbot-skills-found-by-the-bot-they-were-targeting

## Cross-cutting observations

- **MCP support is universal** across the category — every framework above ships first-class MCP. Table stakes, not differentiating.
- **Checkpointing pluggability is the new battleground.** Mature: LangGraph, MS Agent Framework, Claude Agent SDK, Pydantic AI graph, LlamaIndex Workflows. Early/none: CrewAI (`@persist` "may change"), AutoGen, Semantic Kernel.
- **Primitive-shape divergence is wide.** Typed state graph (LangGraph, MS AF) vs. function-decorated + optional graph (Pydantic AI) vs. event-step (LlamaIndex) vs. conversation/crew (AutoGen/AG2/CrewAI) vs. Kernel+plugins (SK) vs. subprocess CLI (Claude Agent SDK) vs. opinionated state machine (Hermes). Each implies different trust-boundary mechanics.
- **Vendor-locked SDKs are well-engineered foils.** Claude Agent SDK has the most mature checkpointing and best tool annotations but is Claude-only; OpenAI Agents SDK is OpenAI-tilted via Traces and Sandbox Agents. Pydantic AI and LangGraph are the cleanest provider-agnostic options.
- **None natively implement CaMeL-style plan/execute trifecta partitioning.** That remains a Donna build regardless of substrate (see `07-security.md`).

## Substrate recommendation

**Lead candidate: LangGraph v1.** Typed state graph maps cleanly to trifecta partitioning; mature pluggable checkpointing with replay/fork; provider-agnostic via LangChain Core; MIT; v1 advertises zero-breaking-change stability — the conservative choice and not a Microsoft consolidation target (`raw/03-agent-frameworks.md`). **Runner-up: Pydantic AI.** Function-decorated agents + `pydantic-graph` + structured-output validation with retry fit Donna's labeled-fact-vs-labeled-inference stance especially well; trade-off is its agent-first ergonomics give less explicit graph control than LangGraph (`raw/03a-frameworks.md`). **Borrow regardless of substrate:** Hermes Agent's MCP-hygiene primitives — env filter, OSV scan, allowlist, error sanitisation (`raw/03b-frameworks.md`). **Rule out as substrate:** AutoGen/AG2/Semantic Kernel (governance + forced-migration risk); Claude Agent SDK and OpenAI Agents SDK (single-vendor lock).
