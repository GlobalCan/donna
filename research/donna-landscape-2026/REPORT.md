# Donna v0.5 — Personal-AI Landscape Synthesis

> Synthesis of `research/donna-landscape-2026/digests/01–08`. Date: 2026-04-25. Currency: anything pre-April-2025 is flagged in the digests. Companion artifacts: per-category digests under `digests/`, raw research under `raw/`, brief at `DONNA_CONTEXT.md`.

## 0. Executive summary

The April-2026 personal-AI landscape has consolidated around three distinct centres of gravity, none of which is Donna's design centre. Cloud-only consumer assistants (Poke, Friend, Day.ai, Sana, Mem.ai, NotebookLM) drift toward enterprise revenue and opaque architecture; OSS chat UIs (LibreChat, AnythingLLM, Open WebUI, Onyx) ship MCP as table stakes but treat memory as an afterthought; agent frameworks (LangGraph v1, Pydantic AI, Claude Agent SDK, MS Agent Framework) are converging on pluggable per-step checkpointing while Microsoft's lineage forces a second migration in 18 months. Memory systems split clean: bitemporal stores (Zep/Graphiti, mnemostack) versus agent-rewrite layers (Letta, Mem0). GraphRAG remains a cost-vs-incrementality trade with no native bitemporal model. Security consensus is settled and uncomfortable: prompt injection is unfixable at the model layer (NCSC Dec 2025), only architectural containment gives provable guarantees (DeepMind CaMeL), and the OpenClaw / ClawHavoc skill-marketplace post-mortem (Jan–Feb 2026) is the proof case for "default-trust plugin store = supply-chain malware vector."

Donna's design tenets — solo-operator self-host, oracle-not-scholar with labeled extrapolation, first-class bitemporal memory, sandboxed tool use, validation surface — sit in the gaps between these centres. Four open lanes survive scrutiny. (i) Inline labeled-extrapolation UI is empty; rationalist epistemic-status headers (Alexander, Appleton, Krycho) and the OpenAI Model Spec (2025-12-18) endorse the stance, but no shipped assistant renders it inline. (ii) Bitemporal memory plus claim-level provenance is unmet across both the memory and GraphRAG categories; Cognee's `temporal_cognify` is the closest event-graph approximation. (iii) Trifecta-partitioned tool sandboxing (CaMeL P-LLM/Q-LLM split, Hermes-style MCP hygiene) is articulated in research but not in any framework's defaults. (iv) Solo-operator-first ergonomics with operator-owned data store dodge the wearable-acquisition pattern (Humane Feb 2025, Rewind/Limitless Dec 2025) and the cloud-pivot pattern (Zep CE deprecation, Open WebUI license change).

The synthesis recommends building Donna on **LangGraph v1** as substrate (zero-breaking-change v1, pluggable checkpointing, no consolidation risk), borrowing **Hermes Agent's MCP hygiene primitives** wholesale (env filter, OSV scan, allowlist, error sanitisation), adopting **Graphiti's bitemporal fact edges** plus **Mem0's episodic/semantic/procedural taxonomy** for memory, and treating **NotebookLM's passage-citing UX** as the floor for the validation surface — then layering inline labeled-inference markup on top. Critical risks to monitor: classifier filters bypassable by reconstruction attacks (CVE-2025-54794), poisoned-memory persistence (no formal mitigation in literature), forced framework migrations (do not couple to AutoGen/AG2/SK), and the Hermes Pattern A/B framing (UNVERIFIED — not in any primary source). Two raw-research framings need correction: the brief's "OpenClaw Nov-2025 / 300+ skills" should read **ClawHavoc, Jan–Feb 2026, 341→1,184 skills, 346k stars**.

## 2. Category deep dives

### 2a. Commercial personal-AI assistants

The commercial slice is the most heterogeneous and the most cautionary. NotebookLM is the cleanest "scholar" foil to Donna's "oracle" — passage-cited, ~95% citation accuracy, ~13% hallucination vs ChatGPT 40% on the same corpus, and zero ability to extrapolate beyond uploaded sources ([notebooklm.google](https://notebooklm.google/)). Granola is the strongest single architectural precedent: local audio capture, on-device transcription, raw audio discarded, transcripts to AWS US, MCP server out (Feb 2026) and personal+enterprise APIs (Mar 2026) — exactly the "local capture + cloud LLM + MCP server" shape Donna wants for ingest ([granola.ai](https://www.granola.ai)). Personal.ai's memory/context/identity taxonomy (lived/static/emergent) maps onto Donna's episodic/semantic/procedural split, though Personal.ai itself has pivoted from consumer twins to telco edge-AI ([personal.ai/memory](https://www.personal.ai/memory)). Two consolidation events in Dec 2025 collapsed the consumer-AI-hardware category: Meta's acquisition sunset Rewind/Limitless on 2025-12-19 and pulled service from EU/UK/several markets; HP's $116M acqui-hire of Humane closed Feb 28 2025 with all AI Pin units bricked. Both validate Donna's self-host tenet — third-party-controlled "personal AI" is one acquisition away from being deleted ([techcrunch](https://techcrunch.com/2025/02/18/humanes-ai-pin-is-dead-as-hp-buys-startups-assets-for-116m/)).

### 2b. OSS self-hosted assistants

Closest architectural neighbours. Khoj is the only entry shipping first-class long-term memory, but its release notes (`2.0.0-beta.28`) document recurring memory-loading bugs and leaks — proof that a half-implemented memory subsystem is worse than none ([github.com/khoj-ai/khoj](https://github.com/khoj-ai/khoj)). AnythingLLM Desktop and Khoj-Obsidian are the cleanest solo trust-boundary distributions; LibreChat / Onyx / Open WebUI / AnythingLLM-Docker default to multi-tenant auth that solo operators pay for nothing. continue.dev's source-controlled markdown agent specs (`.continue/checks/*.md`, diffable via PR) are the strongest borrowable pattern in the category. Open WebUI's April 2025 license pivot — BSD-3 → custom "Open WebUI License" with mandatory CLA and a 50-user branding wall — is a category-wide trust-erosion event and a warning to pre-commit Donna's license stance ([github.com/open-webui/open-webui](https://github.com/open-webui/open-webui)). PrivateGPT is effectively stale (last release Aug 2024, >18 months); treat as RAG reference, not live competition. MCP support is universal across the category; nobody in OSS surfaces intent-of-claim labelling, leaving Donna's open lane intact.

### 2c. Agent frameworks

The substrate decision is the most consequential architectural choice for v0.5. LangGraph v1 (GA Oct 2025, `1.1.9` Apr 2026) is the lead candidate: typed state graph maps cleanly to trifecta partitioning, mature pluggable checkpointing with replay/fork (SQLite/Postgres), provider-agnostic via LangChain Core, MIT, with v1's explicit zero-breaking-change stability commitment ([docs.langchain.com](https://docs.langchain.com/oss/python/releases/langgraph-v1)). Pydantic AI is runner-up — function-decorated agents plus `pydantic-graph` plus structured-output validation with retry-on-failure fit Donna's labeled-fact-vs-labeled-inference stance especially well ([ai.pydantic.dev](https://ai.pydantic.dev/)). Claude Agent SDK has the most mature checkpointing of all (SessionStore S3/Redis/Postgres adapters, file checkpointing, OTel since 0.1.60) but locks Donna to Anthropic. Hermes Agent is a *product* not a substrate; its MCP-hygiene primitives (env filter, OSV scan on every npx/uvx spawn, per-server allowlist, sanitised tool errors) are the most defensible currently shipping and should be borrowed regardless of substrate ([hermes-agent docs](https://hermes-agent.nousresearch.com/docs/user-guide/security)). Microsoft's lineage is disqualified: AutoGen entered maintenance mode 2025-09-30; AG2 forked Nov 2024 over governance; Microsoft Agent Framework 1.0 GA'd 2026-04-03 forcing a second migration in 18 months ([devblogs.microsoft.com](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/)).

### 2d. Memory systems

The category split is clean. Bitemporal stores (Zep / Graphiti / mnemostack) attach event-time and ingestion-time to every fact and invalidate rather than delete, supporting "what did the user believe in March?" queries — the exact substrate Donna's temporal-versioning tenet demands. Agent-rewrite stores (Letta, Mem0) rely on the agent to overwrite blocks; functional but incompatible with Donna's oracle-with-provenance posture. Zep is disqualified as a substrate — its OSS Community Edition was deprecated April 2025 with another retirement wave Feb 2026; only Graphiti remains self-hostable from that family ([blog.getzep.com](https://blog.getzep.com/announcing-a-new-direction-for-zeps-open-source-strategy/)). Mem0 is the only product that explicitly names episodic / semantic / procedural / associative memory types and ships REST CRUD for human-in-loop fact correction with a history DB. Recommended composition for Donna: Graphiti's bitemporal fact model + Mem0's taxonomy + Letta-style directly-editable memory blocks. Reject Letta's "agent decides when facts change" — operator-in-the-loop is non-negotiable. mnemostack's missing edit/delete API is its biggest gap.

### 2e. GraphRAG

Cost vs incrementality is the dominant trade. LightRAG and Cognee tie for cheapest laptop runs (embedded JSON-KV / Kùzu, no DB process). HippoRAG 2 has the strongest published evaluation (ICML 2025, beats LightRAG / GraphRAG / RAPTOR on MuSiQue, 2Wiki, HotpotQA, LV-Eval, NQ, PopQA, NarrativeQA — [arXiv 2502.14802](https://arxiv.org/abs/2502.14802)). Microsoft GraphRAG full mode is the canonical cost trap (one-shot whole-corpus community summarisation, every incremental add re-touches communities); LazyGraphRAG was created to fix it but appears mainly integrated into Azure Local rather than the OSS CLI. **None of the six implements true bitemporal fact storage** — Cognee's `temporal_cognify` (event-graph with timestamp edges and time-aware query parsing) is the closest, and Cognee themselves shipped a Graphiti integration acknowledging Graphiti's bitemporal model is more rigorous. Recommendation: layer Graphiti on top of Cognee's event-tagged graph for Donna; do not adopt Microsoft GraphRAG full mode for a personal corpus.

### 2f. Persona / digital-twin

Donna is not a persona product, but the persona category is the closest existing literature on "an AI that speaks beyond its sources." The shipped products split clean: refuse-to-extrapolate (HereAfter, StoryFile — play back recorded answers only, prove users accept "I don't have a recording" as a refusal mode) versus extrapolate-without-markers (Character.ai, Replika, Pi, RightBack — confident first-person generation with no epistemic markers). The Setzer wrongful-death suit settled Jan 7 2026 is the canonical demonstration of what goes wrong when the second design ships to vulnerable users; Character.ai banned under-18 chat platform-wide Nov 25 2025 ([blog.character.ai](https://blog.character.ai/u18-chat-announcement/)). Replika's Feb 2023 ERP removal under Italian DPA pressure and the lifetime-tier sunset mid-2025 are textbook closed-cloud-product trust collapses ([myhusbandthereplika](https://myhusbandthereplika.wordpress.com/2025/04/05/an-open-letter-to-luka/)). The labelled-extrapolation slot is empty in shipped persona products. Closest precedents are written, not built: Maggie Appleton's "Epistemic Disclosure" ([maggieappleton.com](https://maggieappleton.com/epistemic-disclosure)), Scott Alexander's epistemic-status headers, the OpenAI Model Spec 2025-12-18 instructing models to "proceed based on a best guess while loudly calling out the assumption and uncertainty" ([model-spec.openai.com](https://model-spec.openai.com/2025-12-18.html)), and Anthropic's "Calibrated" honesty pillar. Donna's labeled-extrapolation UI is an inline form of these headers — the writing pattern exists, the UI primitive does not.

### 2g. Security and prompt injection

Consensus across NCSC (Dec 2025), Willison's lethal trifecta (Jun 2025), DeepMind CaMeL (arXiv 2503.18813), and Anthropic/OpenAI training defences is settled and uncomfortable: prompt injection is **not fixable at the model layer**; only architectural containment offers provable guarantees. Classifier filtering pushes attack rates from ~86% to ~4.4% (Constitutional Classifiers gen 1) but reconstruction attacks still bypass them (CVE-2025-54794). Browser-use defences cut attacks 23.6%→~1% with the full Opus 4.5 stack but do not reach zero. CaMeL's P-LLM/Q-LLM split with capability-tagged data flow neutralises 67% of AgentDojo attacks **with proof** while solving 77% of benign tasks ([arXiv 2503.18813](https://arxiv.org/abs/2503.18813)). The OpenClaw / ClawHavoc skill-marketplace post-mortem (Jan–Feb 2026, 341→1,184 malicious skills, 346k stars, CVE-2026-22708, Atomic macOS Stealer payloads, weeks of dwell time — [koi.ai](https://www.koi.ai/blog/clawhavoc-341-malicious-clawedbot-skills-found-by-the-bot-they-were-targeting)) is the proof case for "default-trust plugin marketplace = supply-chain malware vector." Concrete primitives Donna must adopt: trifecta partitioning, plan-execute split, capability tokens on every chunk, strict-schema outputs at every internal hop, instruction-hierarchy tags, garak in CI, Hermes-style MCP hygiene, **no** community skill marketplace by default. Hermes Pattern A/B framing in the brief is **UNVERIFIED** — no primary source uses that naming.

### 2h. Validation tools and ingest

Three tools cleanly meet "cheap + API-accessible + solo-tier" and should be Donna's default external calls: **Google Fact Check Tools API** (free, returns canonical schema.org/ClaimReview — `Claim{text, claimant, claimDate, claimReview[]{publisher, textualRating, reviewDate, url}}` — the canonical "has someone fact-checked this already?" lookup, [developers.google.com](https://developers.google.com/fact-check/tools/api)); **Perplexity Sonar** ($1–$15/M, OpenAI-compatible, citations array — primary grounded retrieval); **Kagi FastGPT/Summarizer** (PAYG ~$0.03/1k tokens — URL/document ingest with reference array). NewsGuard, Full Fact, AllSides API, Logically are all B2B-priced and disqualified at solo tier; Ground News has no public API. The default ingest stack for solo self-host is **yt-dlp** (try `--write-auto-subs` first) → **whisper.cpp large-v3-turbo** locally with **whisperx/pyannote** for diarization; fall through to **AssemblyAI Universal-2** ($0.27/hr) when local diarization is too noisy, or **Deepgram Nova-3** ($0.26–$0.46/hr) for sub-1s streaming. The category's most valuable design lesson is the counter-evidence UX ranking: Community Notes' bridging-based ranking ([github.com/twitter/communitynotes](https://github.com/twitter/communitynotes)) > Ground News Blindspot coverage bar > AllSides Headline Roundup > Perplexity Check Sources > NewsGuard Nutrition Label. The translation for Donna: pick the most credible source contradicting the operator's prior reading and state the counter-claim plainly, no hedging — exactly the "oracle, not scholar" payoff. Borrow Full Fact's `Claim{type, checkworthiness, speaker, topic}` schema for in-house claim extraction (no commodity API exists at solo tier; Factmata's 2022 Cision absorption is the cautionary trajectory).

## 3. Architecture decisions — who chose what

Six axes carve up the landscape; on each axis, Donna's tenets pick a side that is currently underpopulated.

**Agent primitive — typed state graph vs role/conversation vs decorator.** LangGraph v1 and MS Agent Framework 1.0 chose typed state graph; Pydantic AI chose function-decorator + optional `pydantic-graph`; LlamaIndex Workflows chose `@step` event-driven; CrewAI / AutoGen / AG2 chose role/crew/group-chat (multi-agent first); Claude Agent SDK chose subprocess-wrapped CLI; Hermes Agent chose an opinionated state machine. The trade is graph control (explicit trust-boundary nodes — LangGraph's strength) vs ergonomics (function-decorator, Pydantic AI). For a single-user oracle with sandboxed tools, graph control wins; conversation/crew is a category mismatch.

**Hosted vs self-host.** Every shipped commercial assistant (Poke, Day.ai, Sana, Personal.ai, Mem.ai, NotebookLM, Character.ai, Replika, Pi) is cloud-only with closed architecture; every viable Donna substrate (LangGraph, Pydantic AI, Hermes Agent, Letta, Mem0, Graphiti, Cognee) is self-hostable OSS. The cautionary middle is "self-hostable until acquired/deprecated": Zep Community Edition deprecated April 2025, Open WebUI license-pivoted April 2025, Rewind sunset Dec 2025, Humane shut Feb 2025. Donna's self-host tenet is the structural hedge.

**Memory persistence shape.** Bitemporal (Zep / Graphiti / mnemostack — fact edges with `valid_from`/`valid_until`, invalidation not deletion, point-in-time queries) vs agent-rewrite (Letta blocks, Mem0 update operations — agent decides when facts change) vs RAG-only (most OSS chat UIs, all six GraphRAG products — chunk retrieval with no fact identity) vs none/per-conversation (LibreChat, AutoGen, OpenAI Agents SDK Sessions ephemeral). Donna's tenets — "Donna must know *when* facts changed" plus "operator can pull on the validation thread" — only fit bitemporal. Recommended composition: Graphiti's bitemporal substrate with Mem0's episodic/semantic/procedural taxonomy laid on top.

**Tool sandboxing primitive.** Explicit defenders ship something concrete: Hermes Agent (env filter to subprocesses, OSV malware scan on every npx/uvx spawn, per-server tool allowlist, sanitised tool errors), Claude Agent SDK (allow/deny lists, `permission_mode`, Pre/PostToolUse hooks, tool annotations like `destructiveHint`/`readOnlyHint`), OpenAI Agents SDK (Sandbox Agents beta — isolation mechanism undocumented), CaMeL research code (P-LLM/Q-LLM split with capability-tagged data flow). The default-permissive camp is everyone else: LangGraph, LlamaIndex, Semantic Kernel, AutoGen, MS Agent Framework all default to in-process tool execution with no sandbox. Donna's threat model (prompt injection is the default state of the open web) lands on the explicit-defender side.

**Provenance grain.** Passage-cited at the assertion level (NotebookLM ~95% citation accuracy on clinical evals; Perplexity Sonar `citations[]`; Kagi FastGPT `references[]`; Google Fact Check Tools API `claimReview[]`); chunk-level (every GraphRAG product); footer-only (LlamaIndex chat-llamaindex demos); domain/publisher level (NewsGuard, AllSides, Ground News); none (Character.ai, Replika, Pi, Friend, most OSS chat UIs). **Claim-level provenance with confidence/bias/retraction metadata is unmet category-wide.** Donna's validation surface needs to combine NotebookLM's passage-citing UX with Full Fact's tagged-sentence claim schema — neither exists in a single shipped product.

**Persona / extrapolation stance.** Refuse-to-extrapolate (NotebookLM, HereAfter AI, StoryFile, classical RAG citation-only patterns); extrapolate-without-markers (Character.ai, Replika, Pi, RightBack, generative deathbots); extrapolate-with-calibrated-markers in *behaviour* (OpenAI Model Spec mandate, Anthropic Constitution's "Calibrated" pillar — model output, not UI primitive); extrapolate-with-structured-per-claim-status-header in *prose* (Scott Alexander's epistemic-status header, Maggie Appleton's "Epistemic Disclosure", LessWrong/EA Forum convention — human-written, not assistant-rendered). The fifth quadrant — extrapolate-with-structured-per-claim-status-header in *an assistant UI* — is empty. That is Donna's defensible niche.

## 6. Donna's defensible niche

The market scan surfaces four open lanes that intersect at a single product: **a self-hosted, solo-operator personal AI that renders labeled extrapolation inline, over a bitemporally-versioned memory store, behind a trifecta-partitioned tool sandbox.** No shipped competitor occupies that intersection in April 2026.

The labeled-extrapolation slot is the most distinctive. Section 3's persona-stance taxonomy ends in an empty fifth quadrant — extrapolate-with-structured-per-claim-status-header *in an assistant UI*. The cultural pattern is well-established (Alexander, Appleton, Krycho), the model-behaviour mandate is official (OpenAI Model Spec 2025-12-18), the academic caveat is logged (Liu et al. arXiv 2505.24778: markers can lie), but no commercial assistant has shipped the inline rendering. NotebookLM proves the validation-surface floor (passage-citing UX with 95% accuracy); HereAfter proves users accept refusal as a third render mode. Donna's UI primitive is the missing top half of that triad: cited / inferred-with-status / refused.

The Discord-first adapter (`src/donna/adapter/discord_adapter.py`, single chat thread, no multi-tenant abstraction) is a deliberate ergonomic narrowing — it sidesteps the team-auth scaffolding that bloats LibreChat, Onyx, and Open WebUI for solo installs. The two-process architecture (read-only bot, exclusive-writer worker, SQLite WAL with `sqlite-vec`) gives operator-owned data store ergonomics that Mem.ai, Personal.ai, NotebookLM, and Granola structurally cannot match.

The validator stack (`src/donna/security/validator.py` for grounded-citation and debate schemas, `src/donna/security/sanitize.py` for output sanitization, `src/donna/security/taint.py` for capability propagation) is the seed of a Donna-original primitive: a `quoted_span` validator that requires every cited claim to carry a source span ID and verbatim-overlap check, with sanitization run twice (once on retrieved content, once on model output) by cheap models. Combined with strict-schema bottlenecks (Section 4 #5) and trifecta partitioning (#4), this is closer to CaMeL's provable containment than any shipping product offers a solo operator.

The defensible product is not "another agent." It is the **only** assistant that both extrapolates and shows its work — across time, across sources, and across the trust boundary between operator and the open web.

## 7. Open questions for the operator

Substrate decision is the largest blocker. **Confirm LangGraph v1 over Pydantic AI** — the report recommends LangGraph for stability and trifecta-friendly graph control; Pydantic AI fits typed structured-output retry better. Either is defensible; pick before Section 4's checkpointing item lands.
